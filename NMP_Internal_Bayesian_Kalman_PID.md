# 空模型内部机制：贝叶斯-卡尔曼-PID 三层协作

> **版本**: 1.0.0  
> **配套架构**: 三层双视角辩证（TDA）架构  
> **定位**: 第一层「先验计算结构（空模型）」内部感知-估计-响应机制  
> **最后更新**: 2026-07-10

---

## 一、核心命题

空模型不是简单的"通过/拦截"二值判断。它是一个**完整的感知-估计-响应闭环**，在 LLM 介入之前，已经完成了对物理世界的状态辨识、数值估计和误差修正。

> **贝叶斯状态机解决"是什么"，卡尔曼滤波器解决"是多少"，PID 控制器解决"怎么做"。**

三者不是并列替代关系，而是**分层协作、逐级精化**。

 现有自动驾驶/机器人系统：
 
✅ 有贝叶斯状态机（感知）

✅ 有卡尔曼滤波（估计）

✅ 有PID/MPC（控制）

❌ 没有LLM（所以没有"校验LLM"的需求）

❌ 没有事实库/判例库（规则硬编码，不可自学习）

❌ 没有"代表使用者"的语义层（不懂"这个用户是谁"）

> *NMP不替代底层控制，仅叠加语义校验层*

---

## 二、三层协作架构

```
含噪观测序列（传感器 / LLM 输出 / 外部信号）
         ↓
┌─────────────────────────────────────────┐
│ 第一层：状态辨识（贝叶斯状态机）           │
│   输出：确定性状态标签                    │
│   "前方是刚性障碍物还是柔性物体？"         │
│   "当前系统处于正常态还是异常态？"         │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│ 第二层：状态估计（卡尔曼滤波器）           │
│   输出：最优状态估计值（去噪后的具体数值）  │
│   "障碍物距离 45m，接近速度 15m/s"        │
│   "系统偏差 0.3σ，趋势收敛"               │
└─────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────┐
│ 第三层：执行响应（PID 控制器）             │
│   输出：确定性控制量                      │
│   "制动踏板下压 30%"                      │
│   "事实库检索阈值上调 0.15"               │
└─────────────────────────────────────────┘
         ↓
状态标签 + 估计值 + 控制量 → 进入 LLM 上下文或触发拦截
```

## 三、各层详解

### 3.1 第一层：贝叶斯状态机（状态辨识）

#### 定位

从**含噪观测序列**中提取**确定性状态标签**。不是估计具体数值，而是回答"当前处于哪个离散状态"。

#### 核心机制

```
观测序列 O₁, O₂, ..., Oₜ
         ↓
P(Sₜ | O₁:ₜ) ∝ P(Oₜ | Sₜ) · Σ P(Sₜ | Sₜ₋₁) · P(Sₜ₋₁ | O₁:ₜ₋₁)
         ↓
状态信念分布 → 最大后验估计 → 确定性标签
```

#### 在空模型中的作用

| 输入 | 输出 | 解决的问题 |
|------|------|-----------|
| 摄像头帧序列（含噪） | "前方物体：刚性/柔性/未知" | 从变化中识别不变的模式 |
| LLM 输出文本序列 | "生成策略：合规/越界/模糊" | 从概率生成中辨识确定性冲突 |
| 传感器时序数据 | "系统状态：正常/异常/临界" | 从波动中提取结构性状态 |

#### 关键特性

- **利用状态转移先验**：知道"柔性物体不会突然变刚性"
- **从变化中识别不变**：观测在变，但状态标签相对稳定
- **输出是离散的、确定的**：不是"可能是刚性"，是"判定为刚性"

> **贝叶斯状态机是空模型的"眼睛"——它不看细节，只看"这是什么"。**

---

### 3.2 第二层：卡尔曼滤波器（状态估计）

#### 定位

在状态标签确定的前提下，**估计该状态的具体数值**。去噪、预测、修正。

#### 核心机制

```
预测步骤：
  x̂ₖ|ₖ₋₁ = F · x̂ₖ₋₁|ₖ₋₁ + B · uₖ
  Pₖ|ₖ₋₁ = F · Pₖ₋₁|ₖ₋₁ · Fᵀ + Q

更新步骤：
  Kₖ = Pₖ|ₖ₋₁ · Hᵀ · (H · Pₖ|ₖ₋₁ · Hᵀ + R)⁻¹
  x̂ₖ|ₖ = x̂ₖ|ₖ₋₁ + Kₖ · (zₖ - H · x̂ₖ|ₖ₋₁)
  Pₖ|ₖ = (I - Kₖ · H) · Pₖ|ₖ₋₁
```

#### 在空模型中的作用

| 状态标签 | 估计对象 | 输出示例 |
|---------|---------|---------|
| "刚性障碍物" | 距离、速度、质量 | "距离 45m，接近速度 15m/s，估计质量 2.5kg" |
| "系统异常" | 偏差程度、收敛趋势 | "偏差 0.3σ，趋势收敛，预计 3 个周期恢复" |
| "LLM 越界" | 冲突强度、置信度 | "语义冲突度 0.87，事实匹配度 0.12" |

#### 关键特性

- **预测 + 修正**：先根据历史轨迹预测，再用新观测修正
- **动态跟踪变化轨迹**：不是单点估计，是时序估计
- **输出是连续的、去噪的**：从含噪观测中提取最优估计

> **卡尔曼滤波器是空模型的"尺子"——它量出"具体是多少"。**

---

### 3.3 第三层：PID 控制器（执行响应）

#### 定位

根据**目标值与估计值的误差**，计算**确定性控制量**，驱动系统向目标收敛。

#### 核心机制

```
误差 e(t) = 目标值 r(t) - 估计值 y(t)

控制量 u(t) = Kp · e(t) + Ki · ∫e(τ)dτ + Kd · de(t)/dt

┌─────────┐  ┌─────────┐  ┌─────────┐
│ 比例项 P │  │ 积分项 I│  │ 微分项 D │
│ 当前误差 │  │ 历史累积│  │ 变化趋势 │
│ 即时响应 │  │ 消除静差│  │ 抑制振荡 │
└─────────┘  └─────────┘  └─────────┘
```

#### 在空模型中的作用

| 场景 | 目标值 r(t) | 估计值 y(t) | 误差 e(t) | 控制量 u(t) |
|------|------------|------------|----------|------------|
| 自动驾驶 | 安全距离 50m | 当前距离 45m | 5m | 制动踏板下压 30% |
| 事实库检索 | 匹配阈值 0.85 | 当前匹配 0.72 | 0.13 | 检索阈值下调 0.15，扩大召回 |
| 冲突检测 | 冲突阈值 0.90 | 当前冲突 0.87 | 0.03 | 触发二次校验，人工审计介入 |
| 判例库更新 | 判例积累速率 10/天 | 当前速率 3/天 | 7/天 | 放宽反馈触发条件，加速积累 |

#### 关键特性

- **对误差及其导数和积分做瞬时响应**：不是只看现在，是看历史累积和变化趋势
- **输出是确定性的、可执行的**：不是"建议"，是"执行"
- **参数可解释、可调试**：Kp/Ki/Kd 的物理意义明确

> **PID 控制器是空模型的"手"——它做出"怎么做"的确定性动作。**

## 四、三层协作示例：自动驾驶场景

### 场景：前方出现白色塑料袋

| 时间 | 层 | 组件 | 输入 | 输出 |
|------|---|------|------|------|
| t=0 | 感知 | 贝叶斯状态机 | 摄像头帧：白色、飘动、反光 | **状态标签："柔性片状物体"** |
| t=0 | 估计 | 卡尔曼滤波器 | 状态标签 + 激光雷达：距离 30m，速度 5m/s | **估计值：质量 < 0.1kg，碰撞风险极低** |
| t=0 | 响应 | PID 控制器 | 目标：不触发 AEB（柔性物体）- 当前策略：准备急刹 = 误差 | **控制量：抑制 AEB 触发信号，轻收油门 5%** |
| t=1 | 感知 | 贝叶斯状态机 | 新帧：物体被风吹起，形状变化 | **状态标签维持："柔性片状物体"** |
| t=1 | 估计 | 卡尔曼滤波器 | 距离 28m，速度变化不规则 | **估计值：轨迹不可预测，置信度下降，建议人工接管准备** |
| t=1 | 响应 | PID 控制器 | 目标：保持安全 - 当前：不确定性上升 = 误差 | **控制量：降速至 40km/h，开启双闪，准备接管** |

### 关键洞察

- **贝叶斯状态机**识别"这是塑料袋不是石头"——定性判断
- **卡尔曼滤波器**发现"轨迹不可预测，置信度下降"——定量估计
- **PID 控制器**决定"降速 + 双闪 + 准备接管"——确定性响应

> 三层协作，不是替代关系，是**从定性到定量到执行的逐级精化**。

## 五、与 LLM 的关系

| | 空模型（本地） | LLM（云端） |
|--|-------------|------------|
| **状态辨识** | 贝叶斯状态机：确定性标签 | 概率生成：可能性分布 |
| **状态估计** | 卡尔曼滤波器：最优估计值 | 语义描述："大概很远" |
| **执行响应** | PID 控制器：确定性控制量 | 策略建议："建议减速" |
| **本质** | **硬件级确定性** | **软件级概率性** |

> ***空模型负责"必须确定的"，LLM 负责"可以发散的"。***

在 TDA 架构中，空模型的三层感知-估计-响应闭环完成后，才将**状态标签 + 估计值 + 控制量**作为**事实约束包**注入 LLM 上下文。LLM 在此基础上做策略建议，但建议必须经过空模型的**后置校验**（再次运行三层闭环）。

## 六、为什么这三者必须在本地

| 组件 | 延迟要求 | 可靠性要求 | 为什么不能上云 |
|------|---------|-----------|--------------|
| 贝叶斯状态机 | <1ms | 99.999% | 网络抖动导致状态误判 |
| 卡尔曼滤波器 | <2ms | 99.999% | 预测-修正循环需要高频本地数据 |
| PID 控制器 | <2ms | 99.999% | 控制量输出必须实时、确定、不可中断 |

**三者总和 <5ms**，这是空模型校验层延迟指标的来源。

> 不是"为了快而快"，是**确定性控制必须本地执行**——上云的不确定性（网络延迟、服务中断、模型漂移）会破坏控制闭环的稳定性。

## 七、术语对照

| 术语 | 英文 | 在空模型中的角色 |
|------|------|----------------|
| 贝叶斯状态机 | Bayesian State Machine | **状态辨识**：从含噪观测中提取确定性标签 |
| 卡尔曼滤波器 | Kalman Filter | **状态估计**：去噪、预测、修正具体数值 |
| PID 控制器 | PID Controller | **执行响应**：根据误差计算确定性控制量 |
| 感知-估计-响应 | Sense-Estimate-Act | 空模型内部的三层闭环 |
| 状态标签 | State Label | 贝叶斯状态机的离散输出 |
| 最优估计值 | Optimal Estimate | 卡尔曼滤波器的连续输出 |
| 控制量 | Control Output | PID 控制器的确定性输出 |

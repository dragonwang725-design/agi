## 一、现有行业的三层控制实践

当前自动驾驶、智能机器人等关键领域，已经**自发形成了分层控制的工程实践**。这不是理论预设，是物理世界的必然选择——**不同时间尺度的任务，必须由不同响应速度的系统处理**。

### 1.1 三层结构（行业现状）

| 层级 | 名称 | 算力核心 | 响应速度 | 处理事实类型 | 功能定义 | 地位 |
|------|------|---------|---------|------------|---------|------|
| **第一层** | 底层保底安全层（强制策略） | 纯硬件逻辑门、看门狗定时器、AEB 专用芯片 | **微秒级（μs）** | 物理极限事实：最大摩擦力、最小安全距离、电机最大扭矩 | 绝对否决权。物理参数越界→强制切断中高层信号→最大减速度或原地停车 | **物理世界的"宪法"** |
| **第二层** | 中层本地小参数模型（初级策略） | 车载端侧芯片（Orin、地平线），蒸馏小模型（几B~几十B），传统 MPC | **毫秒级（ms）** | 瞬时动态事实：前车距离变化率、车道线曲率、横摆角速度 | 常规执行者。在底层"安全牢笼"内，基于本地化专业知识计算油门/刹车/转向 | **专业知识的"执行手册"** |
| **第三层** | 高层云端大模型（高级策略） | 云端千亿级大模型（GPT/Claude 级别） | **秒级（s）~分钟级** | 稀疏长周期事实：前方3公里事故、副驾乘客身体不适、目的地变更 | 目标修改者。不碰方向盘，推理后下发新的目标值（Setpoint）给中层 | **常识推理的"参谋部"** |

### 1.2 三层的融合博弈机制（行业现状）

这不是简单的"上传下达"，是**互相制约、实时仲裁的动态博弈**：

```
上行通道（事实汇报）：
  第一层 → 第二层："当前摩擦系数 0.3"
  第二层 → 第三层："当前偏移预定路线 200 米"

下行通道（策略下发）：
  第三层 → 第二层："规划新路线走国道"（秒级）
  第二层 → 第一层："前方 200 米右转，减速至 40km/h"（毫秒级）
  第一层 → 第二层："40km/h 过这个弯侧翻风险 87% → 强制降到 30km/h"（微秒级否决）
```

**核心洞察**：LLM（第三层）的"慢"，被第二层（ms）和第一层（μs）完美兜底。即使 LLM 云端崩溃、断网、延迟 30 秒，第二层和第一层依然能保证车辆安全行驶或靠边停车。

## 二、现有体系的成就与边界

### 2.1 成就：物理世界的"宪政雏形"

行业已经**无意识地实践了 TDA 的核心思想**——**不是让一个全能上帝包揽一切，而是让三个各司其职的专家相互制衡**。

| 实践特征 | 对应 TDA 思想 |
|---------|-------------|
| 底层硬约束不可覆盖 | **空模型的最终否决权** |
| 中层只认"既定手册" | **事实库约束下的确定性执行** |
| 高层不碰方向盘 | **LLM 降级为策略建议器** |
| 三层互相否决 | **双视角辩证（主动极 vs 受动极）** |

### 2.2 边界：缺失的"语义校验层"

但现有体系有一个**致命缺口**：

```
现有体系：
  第三层 LLM → "建议改变目的地至最近的医院" → 第二层直接执行
                              ↑
                              └─ 这里缺了一层校验！

NMP 补全后：
  第三层 LLM → "建议改变目的地至最近的医院" → 【空模型校验层】
                                                  ↓
                                      事实库检索：用户病史？医院距离？当前油量？
                                      判例库匹配：上次改道是否出事？
                                                  ↓
                                      校验通过 → 第二层执行
                                      校验拦截 → 重构策略或人工接管
```

| 缺失能力 | 后果 | NMP 补全方案 |
|---------|------|------------|
| **LLM 输出未经事实约束检验** | 第三层可能下发与物理事实、用户事实冲突的目标值 | 空模型校验层：事实库 + 判例库双重检验 |
| **无判例库自学习** | 同类错误重复发生，系统不会"长记性" | 负向积累：一次事故 → 判例 → 全域免疫 |
| **无"代表原则"** | LLM 不区分"这个用户是谁"，给所有人同样的"平均答案" | 元事实库：用户确认的私域事实绑定 |
| **无闭环反馈** | 错误发生后人工复盘，系统本身不进化 | feedback_bounce：自动入判例库，更新阈值 τ |

## 三、TDA/NMP 对现有体系的补全

### 3.1 不是替代，是嵌入

```
现有三层控制体系
├─ 第一层：硬件强制（μs）← 不变
├─ 第二层：端侧模型（ms）← 不变
├─ 第三层：云端 LLM（s）← 不变
└─ 【新增】空模型校验层（<5ms）← NMP 嵌入点
          ↓
    在第三层与第二层之间，插入本地轻量校验
```

**NMP 不触碰现有软硬件，只外挂校验层**。

### 3.2 补全后的四层协作

| 层级 | 原有名称 | NMP 增强后 | 响应速度 |
|------|---------|-----------|---------|
| **第一层** | 硬件强制层 | 硬件强制层 + 空模型物理事实校验 | μs |
| **第二层** | 端侧执行层 | 端侧执行层 + 接收经校验的目标值 | ms |
| **【新增】** | — | **空模型语义校验层**：事实库检索 + 判例库匹配 + 冲突检测 | **<5ms** |
| **第三层** | 云端 LLM | 云端 LLM → 策略建议器（输出经校验后方可下发） | s |

### 3.3 关键场景对比

#### 场景 1：LLM 建议改道医院

| 阶段 | 现有体系 | NMP 增强后 |
|------|---------|-----------|
| LLM 输出 | "建议改道最近医院" | "建议改道最近医院" |
| 校验 | ❌ 无 | ✅ 事实库：用户有心脏病史 + 当前油量仅够 5km + 最近医院 8km → **拦截** |
| 结果 | 半路抛锚，延误救治 | 重构策略："先至最近加油站，再呼叫急救车" |

#### 场景 2：LLM 建议雨天高速行驶

| 阶段 | 现有体系 | NMP 增强后 |
|------|---------|-----------|
| LLM 输出 | "建议保持 120km/h 以节省时间" | "建议保持 120km/h 以节省时间" |
| 校验 | ❌ 无 | ✅ 判例库：`CASE-RAIN-HIGHWAY-007`（雨天 120km/h 侧翻）→ **拦截** |
| 结果 | 侧翻事故 | 重构策略："当前摩擦系数 0.4，建议限速 80km/h" |

#### 场景 3：LLM 建议机器人抓取高温物体

| 阶段 | 现有体系 | NMP 增强后 |
|------|---------|-----------|
| LLM 输出 | "用金属夹抓取电炉上的水壶" | "用金属夹抓取电炉上的水壶" |
| 校验 | ❌ 无 | ✅ 事实库：金属夹导热系数高 + 用户手部有烫伤旧伤 → **拦截** |
| 结果 | 用户二次烫伤 | 重构策略："使用隔热手套 + 关闭电源后等待 5 分钟" |

## 四、理论升华：从"工程分层"到"认知宪政"

### 4.1 现有体系的哲学价值

行业自发的三层控制，已经**无意识地实践了宪政思想**：

| 政治哲学 | 现有体系对应 | TDA 正式化 |
|---------|-----------|-----------|
| **三权分立** | 三层互相否决 | 主动极（LLM）vs 受动极（空模型）的辩证 |
| **宪法至上** | 底层硬约束不可覆盖 | 空模型的最高否决权 |
| **代议制** | 中层按"手册"执行 | LLM 代表使用者立场，非自主判断 |
| **普通法传统** | 无（缺失） | 判例库：经验积累、渐进演化 |

**缺失的"普通法传统"正是 NMP 的核心贡献**——不是预设完备规则，而是让错误成为系统进化的养分。

### 4.2 "有限目的"下的"无限智能"

现有体系已经证明：

> **"判断"（高级策略）可以慢，因为那关乎"去哪"和"为什么"；**  
> **"反应"（初级策略）必须快，因为那关乎"怎么走"；**  
> **"保命"（强制策略）必须瞬发，因为那是物理底线的"事实"。**

NMP/TDA 在此基础上增加：

> **"校验"（语义约束）必须准，因为那关乎"这个建议对这个人在此时此刻是否成立"。**

## 六、结论

> **现有三层控制体系是物理世界的"宪政雏形"——已经证明了分层制衡的必要性。**
>
> **NMP/TDA 不是推翻这个雏形，是补全它缺失的"司法审查层"——让 LLM 的创造性建议，在确定性的事实约束下，才能转化为执行动作。**
>
> **这不是"更好的 LLM"，是"LLM 的宪政框架"。**
